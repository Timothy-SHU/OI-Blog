【ACM-ICPC 2018 南京赛区网络预赛】Set <并查集+Trie合并>

Posted on 2018-10-02

Problem

【ACM-ICPC 2018 南京赛区网络预赛】Set

Time  Limit:  1500  ms\mathrm{Time\;Limit:\;1500\;ms}TimeLimit:1500ms
Memory  Limit:  524288  K\mathrm{Memory\;Limit:\;524288\;K}MemoryLimit:524288K

Description

Shinku is very interested in the set. One day, she got nnn sets, and the ithi^{th}ith number aia_iai​ is in the ithi^{th}ith set. But she doesn’t think it is interesting enough, so she applies mmm magic to these sets. There are three kinds of magic:

  • 1  u  v1\;u\;v1uv: If the uthu^{th}uth and vthv^{th}vth numbers are not in one set, then the Shinku’s magic will merge the set containing the uthu^{th}uth number and the set containing the vthv^{th}vth number.
  • 2  u2\;u2u: Shinku’s magic adds 111 to each number in the set containing the uthu^{th}uth number.
  • 3  u  k  x3\;u\;k\;x3ukx: Shinku can immediately know how many numbers ttt in the set containing the uthu^{th}uth number satisfy t≡x( mod  2k)t\equiv x (\bmod\ 2^k)t≡x(mod 2k) (0≤k≤30,0≤x<2k)(0 \le k\le 30,0\le x<2^k)(0≤k≤30,0≤x<2k).

But unfortunately, for some reason the type 333 magic fails. So Shinku wants you to tell her the answer after every type 333 magic.
Note that there can be multiple numbers with the same value in one set, that is, numbers with the same value will not disappear when merged.

Input

The first line contains two integers n,mn,mn,m (1≤n,m≤6×105)(1 \le n,m \le 6 \times 10^5)(1≤n,m≤6×105), the number of initial sets and the number of the magic.
The second line contains nnn integers. The ithi^{th}ith number aia_iai​ (0≤ai≤109)(0 \le a_i \le 10^9)(0≤ai​≤109) is the number in the ithi^{th}ith set initially.
The next mmm lines describe the sequence of magic. The ithi^{th}ith line describes the ithi^{th}ith magic. Each magic is a magic as described above.

Output

For each type 333 magic, output the answer you are asked to calculate.

Explanation

After the first operation, the numbers are 2,3,42,3,42,3,4, sets are {2,4}\lbrace 2,4 \rbrace{2,4} {3}\lbrace 3 \rbrace{3}
For the second operation, the third number is in {2,4}\lbrace 2,4 \rbrace{2,4}, 2≡0(mod21)2 \equiv 0\pmod {2^1}2≡0(mod21), 4≡0(mod21)4 \equiv 0\pmod {2^1}4≡0(mod21), so the answer is 222.
After the third operation, the numbers are 2,4,42,4,42,4,4, sets are {2,4}\lbrace 2,4 \rbrace{2,4} {4}\lbrace 4 \rbrace{4}
After the forth operation, the numbers are 2,4,42,4,42,4,4, sets are {2,4,4}\lbrace 2,4,4 \rbrace{2,4,4}
For the fifth operation, ,the third number is in {2,4,4}\lbrace 2,4,4 \rbrace{2,4,4}, 2≡0(mod21)2 \equiv 0\pmod {2^1}2≡0(mod21), 4≡0(mod21)4 \equiv 0\pmod {2^1}4≡0(mod21),4≡0(mod21)4 \equiv 0\pmod {2^1}4≡0(mod21), so the answer is 333.

Translation

给定一个长为nnn (n≤5⋅105)(n\le5\cdot10^5)(n≤5⋅105)的序列{a}\lbrace a\rbrace{a} (ai≤109)(a_i\le10^9)(ai​≤109)，起初第iii个数在第iii个集合。
有mmm (m≤5⋅105)(m\le5\cdot10^5)(m≤5⋅105)个操作，分为三类：

  • 1  u  v1\;u\;v1uv 将第uuu个数和第vvv个数所在集合合并
  • 2  u2\;u2u 将第uuu个数所在集合所有数加111
  • 3  u  k  x3\;u\;k\;x3ukx 问uuu所在集合有多少个数模2k2^k2k余xxx (k≤30)(k\le 30)(k≤30)

对于每个333询问，输出答案。

Solution

探索题目条件的特殊性，发现所有询问均是对2k2^k2k取模。
我们将每个数转为二进制后倒置，则询问变为含有xxx转为二进制倒置作为前缀的二进制串的个数，可以用Trie\mathrm{Trie}Trie维护。那么111操作相当于合并两个Trie\mathrm{Trie}Trie，可以用类似线段树合并的做法完成，需要用并查集维护连通性。
对于222操作，由于每个数都倒置了，可以模拟从最后一位向前进位。于是给结点打上标记，每次下传时将标记除222传到下一层，特别地，如果当前层标记是奇数，则需要交换两个子树，因为进位会使000变为111，111变为000。
总时间复杂度O((n+q)log⁡109)O((n+q)\log10^9)O((n+q)log109)。
P.S.\mathrm{P.S.}P.S.貌似对222操作可以暴力向下递归交换子树。
