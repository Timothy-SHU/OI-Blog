计蒜客【NOIP2017模拟3】任性的国王 <线段树+MST>

Posted on 2018-10-22

Problem

【NOIP2017模拟3】任性的国王

Time  Limit:  1000  ms\mathrm{Time\;Limit:\;1000\;ms}TimeLimit:1000ms
Memory  Limit:  262144  K\mathrm{Memory\;Limit:\;262144\;K}MemoryLimit:262144K

Description

X 国的地图可以被看作一个两行 nnn 列的网格状图。现在 X 国需要修建铁路，然而该国的国王非常小气，他只想保证位于某两列之间的所有城市互相可以到达就行了，在此基础上，他希望所花费的代价最小。
铁路可以建在任何两个相邻的点之间，使他们可以互相到达。可以作为工作人员，你已经整理出了为每一对相邻城市架设铁路所需要的花费。你需要准备好回答国王如下形式的问题。
对于 (i,j)(i,j)(i,j)：当前情况下，使第 iii 列到第 jjj 列之间的所有城市连通的最小代价是多少（列下标从 111 开始）？注意不能用其他列的城市。
然而你还有更大的困难，随着时间变化，使用某些边作为铁路的代价会发生改变，你必须有效率地处理这些变化。

Input

第一行两个正整数 n,mn,mn,m，表示该国有 222 行 nnn 列以及 mmm 个询问或者操作。
第二行 3n−23n-23n−2 个数，前 n−1n-1n−1 个数依次表示在第一行的 n−1n-1n−1 条边上修建铁路的代价。
接下来 n−1n−1n−1 个数，依次表示在第二行的 n−1n-1n−1 条边上修建铁路的代价。
最后 nnn 个数，依次表示在第 111 列到第 nnn 列的边上修建铁路的代价。
接下来 mmm 行的输入具有如下形式，K,S,TK,S,TK,S,T，其中

  • 若 K=1K=1K=1，则表示询问当前状态下，使所有第 SSS 列到第 TTT 列之间的城市连通需要的最小代价。
  • 若 K=2K=2K=2，则表示位于第 111 行第 SSS 列的点到第 111 行第 S+1S+1S+1 列的点之间的边上修建铁路的代价变为 TTT。
  • 若 K=3K=3K=3，则表示位于第 222 行第 SSS 列的点和第 222 行第 S+1S+1S+1 列的点之间的边上修建铁路的代价变为 TTT。
  • 若 K=4K=4K=4，则表示第 SSS 列的边上修建铁路的代价变为 TTT。

Output

依次对每个询问，用一行输出相应的答案。

Constraint

对于 30%30\%30% 的数据：n,m≤2000n,m\le 2000n,m≤2000。
另有 30%30\%30% 的数据：所有竖边的代价为 000 且永不改变。
对于全部数据：n,m≤105n,m \le 10^5n,m≤105。
所有输入和输出数据保证合法，且不超过 231−12^{31}-1231−1。

标签：线段树 MST

Solution

线段树维护特殊的网格图MST\mathrm{MST}MST。

对于每个区间[s,t][s,t][s,t]，维护四个值mi[1/0][1/0]mi[1/0][1/0]mi[1/0][1/0]，分别代表第sss列的竖边选/不选且第t+1t+1t+1列竖边选/不选的情况下，将第sss个网格到第ttt个网格中所有点连起来的最小代价。
用ru[i],rd[i]ru[i],rd[i]ru[i],rd[i]表示第iii个网格上/下的边，用c[i]c[i]c[i]表示第iii列的竖边，那么第iii个网格相邻的两条竖边为c[i],c[i+1]c[i],c[i+1]c[i],c[i+1]。
对于长度为111的区间[i,i][i,i][i,i]，直接赋初值：

mi[0][0]=∞mi[1][0]=ru[i]+rd[i]+c[i]mi[0][1]=ru[i]+rd[i]+c[i+1]mi[1][1]=min⁡(ru[i],rd[i])+c[i]+c[i+1]mi[0][0]=\infty\\ mi[1][0]= ru[i]+rd[i]+c[i]\\ mi[0][1]=ru[i]+rd[i]+c[i+1]\\ mi[1][1]=\min(ru[i],rd[i])+c[i]+c[i+1]\\ mi[0][0]=∞mi[1][0]=ru[i]+rd[i]+c[i]mi[0][1]=ru[i]+rd[i]+c[i+1]mi[1][1]=min(ru[i],rd[i])+c[i]+c[i+1]

对于其余区间，需要从两个子区间合并而来。设左右区间为x,yx,yx,y，则两区间相交处的竖边为c[x.t+1]c[x.t+1]c[x.t+1]。
那么对于此区间的mi[i][j]mi[i][j]mi[i][j]，一定从x.mi[i][?]x.mi[i][?]x.mi[i][?]和y.mi[?][j]y.mi[?][j]y.mi[?][j]合并而来。如果用x.mi[i][0]x.mi[i][0]x.mi[i][0]和y.mi[0][j]y.mi[0][j]y.mi[0][j]合并，其合并出的所有情况的MST\mathrm{MST}MST权值无法直接算出，但其一定能由其余的三种合并方式得到，于是可以不用管这种合并方式。注意合并时第x.t+1x.t+1x.t+1列的两条边会在两棵生成树中均联通，所以需要删去其中一棵中连接两点的边，即减去c[s.t+1]c[s.t+1]c[s.t+1]。
由此，我们用线段树维护这个信息，对于修改，直接到该位置重新赋初值然后维护其祖先的信息即可。
时间复杂度O(nlog⁡n)\mathcal{O}(n\log{n})O(nlogn)。
