BZOJ1071【SCOI2007】组队 <双指针>

Posted on 2018-03-27

Problem

【SCOI2007】组队

Time  Limit:  3  Sec\mathrm{Time\;Limit:\;3\;Sec}TimeLimit:3Sec
Memory  Limit:  128  MB\mathrm{Memory\;Limit:\;128\;MB}MemoryLimit:128MB

Description

NBANBANBA每年都有球员选秀环节。通常用速度和身高两项数据来衡量一个篮球运动员的基本素质。假如一支球队里速度最慢的球员速度为minVmin_VminV​，身高最矮的球员高度为minHmin_HminH​，那么这支球队的所有队员都应该满足：A×(height−minH)+B×(speed−minV)≤CA\times(height-min_H)+B\times(speed-min_V)\le CA×(height−minH​)+B×(speed−minV​)≤C。其中AAA和B,CB,CB,C为给定的经验值。这个式子很容易理解，如果一个球队的球员速度和身高差距太大，会造成配合的不协调。 请问作为球队管理层的你，在NNN名选秀球员中，最多能有多少名符合条件的候选球员。

Input

第一行四个数N,A,B,CN,A,B,CN,A,B,C。
下接NNN行每行两个数描述一个球员的heightheightheight和speedspeedspeed。

Output

最多候选球员数目。

HINT

N≤5000N\le5000N≤5000，heightheightheight和speedspeedspeed不大于100001000010000。A,B,CA,B,CA,B,C在长整型以内。

标签：双指针

Solution

挺神的题，两个序列上双指针的操作挺奇葩。

对于每个队员，令s=v+hs=v+hs=v+h，将队员数组AAA同时复制到BBB中，分别排序。AAA按照hhh从小到大排序，BBB按照sss从小到大排序。
首先按任意顺序枚举vvv的最小值大小，不妨直接再AAA中枚举。这时定义左指针和右指针lll和rrr，只是与普通双指针不同的是，lll指向的是AAA中的元素，rrr指向的是BBB中的元素，可以理解为AAA中的右指针是不动的，BBB中的左指针是不动的。这样精妙设计是考虑到AAA中递增的hhh值如果不满足要求，即小于枚举到的HHH值，小于的部分一定是从前面开始的；同样地，BBB中递增的sss值如果不满足要求，即大于A×minH+B×minV+CA\times min_H+B\times min_V+CA×minH​+B×minV​+C，大于的部分一定是从后面开始的。这样对每个数组跑“单指针”即可得到可行最大子段，更新答案即可。
