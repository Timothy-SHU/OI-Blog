BZOJ3289 Mato的文件管理 <莫队+树状数组>

Posted on 2018-04-03

Problem

Mato的文件管理

Time  Limit:  40  Sec\mathrm{Time\;Limit:\;40\;Sec}TimeLimit:40Sec
Memory  Limit:  128  MB\mathrm{Memory\;Limit:\;128\;MB}MemoryLimit:128MB

Description

Mato\mathrm{Mato}Mato同学从各路神犇以各种方式收集了许多资料，这些资料一共有nnn份，每份有一个大小和一个编号。为了防止他人偷拷，这些资料都是加密过的，只能用Mato\mathrm{Mato}Mato自己写的程序才能访问。Mato\mathrm{Mato}Mato每天随机选一个区间[l,r][l,r][l,r]，他今天就看编号在此区间内的这些资料。Mato\mathrm{Mato}Mato有一个习惯，他总是从文件大小从小到大看资料。他先把要看的文件按编号顺序依次拷贝出来，再用他写的排序程序给文件大小排序。排序程序可以在111单位时间内交换222个相邻的文件（因为加密需要，不能随机访问）。Mato\mathrm{Mato}Mato想要使文件交换次数最小，你能告诉他每天需要交换多少次吗？

Input

第一行一个正整数nnn，表示Mato\mathrm{Mato}Mato的资料份数。
第二行由空格隔开的nnn个正整数，第iii个表示编号为iii的资料的大小。
第三行一个正整数qqq，表示Mato\mathrm{Mato}Mato会看几天资料。
之后qqq行每行两个正整数l,rl,rl,r，表示Mato\mathrm{Mato}Mato这天看[l,r][l,r][l,r]区间的文件。

Output

qqq行，每行一个正整数，表示Mato\mathrm{Mato}Mato这天需要交换的次数。

Sample Output

Hint

样例解释
第一天，Mato\mathrm{Mato}Mato不需要交换
第二天，Mato\mathrm{Mato}Mato可以把222号交换222次移到最后。
数据规模和约定
n,q≤5×104n,q\le5\times10^4n,q≤5×104

Source

By taorunz

标签：莫队 树状数组

Solution

最小交换次数=区间逆序对数最小交换次数=区间逆序对数最小交换次数=区间逆序对数
因此此问题等价于求区间逆序对数，傻逼莫队水过。
离线下所有询问，每次加入或删除一个数用树状数组维护，计算会增加或减少多少对逆序对，更新答案即可。
