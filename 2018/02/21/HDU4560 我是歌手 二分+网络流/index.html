HDU4560 我是歌手 <二分+网络流>

Posted on 2018-02-21

Problem

我是歌手

Time  Limit:  2000  MS\mathrm{Time\;Limit:\;2000\;MS}TimeLimit:2000MS
Memory  Limit:  32768  KB\mathrm{Memory\;Limit:\;32768\;KB}MemoryLimit:32768KB

Description

201320132013年一开始，一档音乐节目“我是歌手”就惊艳了大家一回。闲话少说，现在，你成为了这档节目的总导演，你的任务很简单，安排每一期节目的内容。
现在有NNN个歌手，MMM种歌曲流派（RockRockRock，PopPopPop之类），每个歌手都有自己擅长的流派领域，这些资料都已整理。你的工作是，安排尽可能多场的演唱比赛。每一场比赛所有歌手都必须上场，为了提高收视率，每个人演唱的歌曲类型不能相同，即便一些歌手要被迫选择一些他们不擅长的。同时，为了展现全面性，在不同的演唱比赛上，每个歌手都会安排不同的歌曲流派。
但是问题是，对于任何一个歌曲流派的歌迷，如果超过KKK个不擅长的歌手演唱了这种歌曲，他们就会表示不满，比如，发一些宣泄不满的帖子微博，为了表示观点挑起事端等等。你当然不希望这些事情与你的节目有关，在这个前提下，你可以任意安排尽可能多的比赛场次。

Input

输入第一行为TTT，表示有TTT组测试数据。
每组数据以四个数字NNN，MMM，LLL，KKK开始。LLL表示有LLL组擅长关系，接下来的LLL行，每一行有两个数字AiA_iAi​，BiB_iBi​，表示歌手AiA_iAi​擅长BiB_iBi​类型的歌曲。

Output

对每组数据，先输出为第几组数据，然后输出最多比赛场次。

Solution

挺好的一道二分+网络流二分+网络流二分+网络流套路建模题。

二分最多能安排的场次，得到当前答案进行判定。

首先，如果保证每个人唱每种流派最多仅一次，并且nnn个流派均会被唱至少tanstanstans次，那么一定能找到一组解。这样就可以用最大流判定。
由于有“每个流派仅能有kkk个不擅长的选手唱”，我们需要把每个流派又擅长的选手唱和不擅长的选手唱分开，而限制不擅长的选手的数量。可将每个流派拆成两个点，即第iii个流派StyleiStyle_iStylei​拆成StyleiStyle_iStylei​和Stylei′Style_i'Stylei′​。从StyleiStyle_iStylei​向擅长的选手连边，从Stylei′Style_i'Stylei′​向不擅长的选手连边，而由于从源点向StyleiStyle_iStylei​连有tanstanstans的边，因而所有可用流量都在StyleiStyle_iStylei​中，Stylei′Style_i'Stylei′​的流量需要限制，故要在StyleiStyle_iStylei​和Stylei′Style_i'Stylei′​间连KKK的边来限制不擅长选手的数量。

建模总述：

S→Stylei(i∈[1,m]):Capacity=tansStylei→Stylei′(i∈[1,m]):Capacity=KStylei→Competitorj(Competitorj is good at Stylei):Capacity=1Stylei′→Competitorj(Competitorj is not good at Stylei):Capacity=1Competitorj→T(j∈[1,n]):Capacity=tans\begin{aligned} S &\to Style_i(i\in [1,m]):Capacity=tans\\ Style_i &\to Style_i'(i\in [1,m]):Capacity=K\\ Style_i &\to Competitor_j(Competitor_j\ is\ good\ at\ Style_i):Capacity=1\\ Style_i' &\to Competitor_j(Competitor_j\ is\ not\ good\ at\ Style_i):Capacity = 1\\ Competitor_j &\to T(j\in [1,n]):Capacity=tans\\ \end{aligned} SStylei​Stylei​Stylei′​Competitorj​​→Stylei​(i∈[1,m]):Capacity=tans→Stylei′​(i∈[1,m]):Capacity=K→Competitorj​(Competitorj​ is good at Stylei​):Capacity=1→Competitorj​(Competitorj​ is not good at Stylei​):Capacity=1→T(j∈[1,n]):Capacity=tans​

建模后跑最大流，看是否有MaxFlow=tans×nMaxFlow=tans\times nMaxFlow=tans×n即可。
