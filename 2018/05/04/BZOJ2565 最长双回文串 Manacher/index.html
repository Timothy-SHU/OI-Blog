BZOJ2565 最长双回文串 < Manacher >

Posted on 2018-05-04

Problem

最长双回文串

Time  Limit:  10  Sec\mathrm{Time\;Limit:\;10\;Sec}TimeLimit:10Sec
Memory  Limit:  128  MB\mathrm{Memory\;Limit:\;128\;MB}MemoryLimit:128MB

Description

顺序和逆序读起来完全一样的串叫做回文串。比如acbca是回文串，而abc不是（abc的顺序为abc，逆序为cba，不相同）。
输入长度为nnn的串SSS，求SSS的最长双回文子串TTT，即可将TTT分为两部分X,Y  (∣X∣,∣Y∣≥1)X,Y\;(|X|,|Y|\ge1)X,Y(∣X∣,∣Y∣≥1)且XXX和YYY都是回文串。

Input

一行由小写英文字母组成的字符串SSS。

Output

一行一个整数，表示最长双回文子串的长度。

Explanation

从第二个字符开始的字符串aacaabbacabb可分为aacaa与bbacabb两部分，且两者都是回文串。

Solution

Manacher\mathrm{Manacher}Manacher上稍加变化。

首先跑Manacher\mathrm{Manacher}Manacher得到以每个位置为中心的回文串最大长度。然后考虑计算出{L}\lbrace L\rbrace{L}和{R}\lbrace R\rbrace{R}，分别表示以每个位置为终点和起点的最长回文串的中心点位置。如此枚举每个位置作为中间断点打擂得到最长双回文字串。

那么如何计算{L}\lbrace L\rbrace{L}和{R}\lbrace R\rbrace{R}呢？
对于位置iii，其最长回文串半径长为rir_iri​，那么区间[i,i+ri)[i,i+r_i)[i,i+ri​)中的所有位置都可以作为这个串的右端点（终点）。于是这些位置的LLL值一定不大于iii，这是因为LLL值越小，回文串越长，这样更优。所以从前往后枚举iii，如果[i,i+ri)[i,i+r_i)[i,i+ri​)中的某个点的LLL值在前面没有确定到，那么这个点的LLL值一定最小为iii。发现这样的点一定在一个区间中，所以可以记录每次更新LLL更新到的位置，即可O(n)O(n)O(n)扫一遍得到{L}\lbrace L\rbrace{L}。反着这样扫一遍即可得到{R}\lbrace R\rbrace{R}。
