BZOJ5087 Polycomp < bitset+分块 >

Posted on 2018-05-20 Edited on 2018-05-22

Problem

Polycomp

$\mathrm{Time;Limit:;40;Sec}$
$\mathrm{Memory;Limit:;256;MB}$

Description

你有三个系数为$0,1$的多项式$f(x),g(x),h(x)$
求$f(g(x))\mod{h(x)}$
为方便起见，将答案多项式所有系数对$2$取模输出即可
如果$f(x)=\sum_{k=1}^{n} a_k\cdot x^k$，则$f(g(x))=\sum_{k=1}^{n}a_k\cdot g(x)^k$

Input

一共三行，每行一个多项式，分别为$f,g,h$
对于一个多项式$P$，描述为$n$个整数$P_0,P_1,\cdots,P_n$，其中$P_i$为$0$或$1$，$P(x)=P_0+P_1\cdot x+\cdots+P_n\cdot x_n$

Output

用同样的格式输出答案多项式
如果答案为$0$，输出0 0

Sample Input

Sample Output

HINT

记$n$表示多项式最高项的次数，$n\le4000$

Source

By clj

标签：bitset 分块

Solution

陈老师神题。

直接用数组模拟乘除过程，可以做到$O(n^3)$，若用bitset维护，即可做到$O(\frac{n^3}{32})$。然而这样还是会$\mathrm{TLE}$。

考虑类似分块的想法，将$f$的每十位分成一段，预处理$2^{10}$种情况对应的和，然后每次十位十位地加，即可做到$O(\frac{n^3}{320})$，时间复杂度可以接受。
根据vanilla的调参，貌似这种先预处理的方法在块大小为$16$时最快。

OwenOwl有更快的方法，即先不预处理出所有和，最后加的时候再根据每块情况计算，这样块大小可以更大，情况数更少，会更快。
