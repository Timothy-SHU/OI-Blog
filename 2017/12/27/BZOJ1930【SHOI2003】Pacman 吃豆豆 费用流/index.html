BZOJ1930【SHOI2003】Pacman 吃豆豆 <费用流>

Posted on 2017-12-27

Problem

【SHOI2003】Pacman 吃豆豆

Time  Limit:  10  Sec\mathrm{Time\;Limit:\;10\;Sec}TimeLimit:10Sec
Memory  Limit:  64  MB\mathrm{Memory\;Limit:\;64\;MB}MemoryLimit:64MB

Description

两个PACMAN\mathrm{PACMAN}PACMAN吃豆豆。一开始的时候，PACMAN\mathrm{PACMAN}PACMAN都在坐标原点的左下方，豆豆都在右上方。PACMAN\mathrm{PACMAN}PACMAN走到豆豆处就会吃掉它。PACMAN\mathrm{PACMAN}PACMAN行走的路线很奇怪，只能向右走或者向上走，他们行走的路线不可以相交。 请你帮这两个PACMAN\mathrm{PACMAN}PACMAN计算一下，他们俩加起来最多能吃掉多少豆豆。

Input

第一行为一个整数NNN，表示豆豆的数目。 接下来 NNN 行，每行一对正整数，表示第iii个豆豆的坐标。任意两个豆豆的坐标都不会重合。

Output

仅有一行包含一个整数，即两个PACMAN\mathrm{PACMAN}PACMAN加起来最多能吃掉的豆豆数量

Sample Output

HINT

N≤2000N \le 2000N≤2000

Solution

貌似是可以用DP\mathrm{DP}DP肝的。
费用流建模细节挺多。
首先可以发现不相交时废话。若穿过则互换名字即可。
由于点数很多，所以不能两两连边，发现只需要把按xxx和yyy排序后相邻两点连边即可。
首先需要限制每个点的流量，需要将每个点拆成两个点，连边限流。
这里由于两条线可以经过同一个点，但贡献只算一个，则需要连两条边，流量均为111，而费用则是一条为111另一条为000。
注意源点也需要限制222的流量，即需要把源点拆成两个点，中间连流量222费用000的边。
相邻两点间连流量为222费用为000的边。
总结建图：

  • 源点拆成两个点SSS和SSSSSS
    S→SSS\to SSS→SS 流量222 费用000
  • 把每个点拆成iii和i′i'i′
    SS→iSS\to iSS→i 流量222 费用000
    i′→Ti'\to Ti′→T 流量222 费用000
    i→i′i\to i'i→i′ 流量111 费用111
    i→i′i\to i'i→i′ 流量111 费用000
  • 可连边的相邻两点iii和jjj间有
    i′→ji'\to ji′→j 流量222 费用000

最后跑大费流即可。
