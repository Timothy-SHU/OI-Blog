BZOJ3226【SDOI2008】校门外的区间 <线段树>

Posted on 2017-09-22

Problem

【SDOI2008】校门外的区间

Time  Limit:  10  Sec\mathrm{Time\;Limit:\;10\;Sec}TimeLimit:10Sec
Memory  Limit:  128  MB\mathrm{Memory\;Limit:\;128\;MB}MemoryLimit:128MB

Description

受校门外的树这道经典问题的启发，AAA君根据基本的离散数学的知识，抽象出555种运算维护集合SSS(SSS初始为空)并最终输出SSS。现在，请你完成这道校门外的树之难度增强版——校门外的区间。
555种运算如下：

编号 表示格式 数学表示
111 U    TU\;\;TUT S∪TS\cup TS∪T
222 I    TI\;\;TIT S∩TS\cap TS∩T
333 D    TD\;\;TDT S−TS-TS−T
444 C    TC\;\;TCT S+TS+TS+T
555 S    TS\;\;TST S⊗TS\otimes TS⊗T

基本集合运算如下：

运算 结果
A∪BA\cup BA∪B {x∣x∈A∣∣x∈B}\lbrace x\mid x\in A\mid\mid x\in B\rbrace{x∣x∈A∣∣x∈B}
A∩BA\cap BA∩B {x∣x∈A    &&    x∈B}\lbrace x\mid x\in A\;\;\&\&\;\;x\in B\rbrace{x∣x∈A&&x∈B}
A−BA-BA−B {x∣x∈A    &&    x∉B}\lbrace x\mid x\in A\;\;\&\&\;\;x\notin B\rbrace{x∣x∈A&&x∈/B}
A⊗BA\otimes BA⊗B (A−B)∪(B−A)(A-B)\cup(B-A)(A−B)∪(B−A)

Input

输入共MMM行。
每行的格式为XXX TTT，用一个空格隔开，XXX表示运算的种类，TTT为一个区间(区间用(a,b),(a,b],[a,b),[a,b](a,b), (a,b], [a,b), [a,b](a,b),(a,b],[a,b),[a,b]表示)。

Output

共一行，即集合SSS，每个区间后面带一个空格。若SSS为空则输出emptyemptyempty setsetset。

Sample Output

HINT

对于 100%100\%100% 的数据，0≤a≤b≤655350\le a\le b\le 655350≤a≤b≤65535，1≤M≤700001\le M\le 700001≤M≤70000

标签：线段树

Solution

假设只有闭区间，对于每个数，标记其为111还是000。
四种运算对应如下（modifymodifymodify为区间修改，reversereversereverse为区间取反）：

  • UUU a,b⇒modify(a,b,1)a,b\Rightarrow modify(a, b, 1)a,b⇒modify(a,b,1)
  • DDD a,b⇒modify(a,b,0)a,b\Rightarrow modify(a, b, 0)a,b⇒modify(a,b,0)
  • SSS a,b⇒reverse(a,b)a,b\Rightarrow reverse(a, b)a,b⇒reverse(a,b)
  • CCC a,b⇒modify(0,a−1,0),modify(b+1,n,0),reverse(a,b)a,b\Rightarrow modify(0, a-1, 0), modify(b+1, n, 0), reverse(a, b)a,b⇒modify(0,a−1,0),modify(b+1,n,0),reverse(a,b)
  • III a,b⇒modify(0,a−1,0),modify(b+1,n,0)a,b\Rightarrow modify(0, a-1, 0), modify(b+1, n, 0)a,b⇒modify(0,a−1,0),modify(b+1,n,0)

再考虑加入开区间，开区间看作对应.5.5.5的闭区间。即将(4,7)(4, 7)(4,7)看作[4.5,6.5][4.5, 6.5][4.5,6.5]。
为了存.5.5.5的小数，我们将所有区间均乘222，即(4,7)(4, 7)(4,7)变为[9,13][9, 13][9,13]。
又考虑到有000的区间，因而对于所有区间两端点再加222，即(4,7)(4,7)(4,7)变为[11,15][11, 15][11,15]。
最后用线段树维护即可。
对于输出，O(nlog⁡(n))O(n\log(n))O(nlog(n))扫一遍，找出每个数是111还是000，然后合并区间，双指针跑。
本题细节较多，写的时候得小心。
