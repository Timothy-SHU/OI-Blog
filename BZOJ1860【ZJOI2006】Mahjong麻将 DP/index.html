Problem

【ZJOI2006】Mahjong麻将

Time  Limit:  1  Sec\mathrm{Time\;Limit: \;1\;Sec}TimeLimit:1Sec
Memory  Limit:  64  MB\mathrm{Memory\;Limit:\;64\;MB}MemoryLimit:64MB

Description

很多人都知道玩麻将，当然也有人不知道，呵呵，不要紧，我在这里简要地介绍一下麻将规则：
普通麻将有砣、索、万三种类型的牌，每种牌有1∼91\sim 91∼9个数字，其中相同的牌每个有四张，例如1砣∼9砣1砣\sim 9砣1砣∼9砣，1索∼9索1索\sim 9索1索∼9索，1万∼9万1万\sim 9万1万∼9万各444张，所以共36×3=10836\times 3=10836×3=108张牌。胡牌时每人有141414张牌，其中只要某人手里有若干句话（就是同种类型的牌连续三张或同种牌三张），另外再加上一对，即可胡牌。当然如果全是对，叫七小对，也可以胡牌。
要判断某人是否胡牌，显然一个弱智的算法就行了，某中学信息学小组超级麻将迷想了想，决定将普通麻将改造成超级麻将。
所谓超级麻将没有了砣、索、万的区分，每种牌上的数字可以是1∼1001\sim 1001∼100而每种数字的牌各有100100100张。另外特别自由的是，玩牌的人手里想拿多少张牌都可以，好刺激哦！
刺激归刺激，但是拿多了怎么胡牌呢？
超级麻将规定只要一个人手里拿的牌是若干句话（三个连续数字的牌各一张组成一句话，三张或四张同样数字的牌也算一句话），再加上一对相同的牌，就算胡了。
作为信息学竞赛选手的你，麻烦你给这位超级麻将迷编个程序，判断能否胡牌。

Input

第一行一个整数NNN（N≤100N\le 100N≤100），表示玩了NNN次超级麻将。 接下来NNN行，每行100100100个数a1,a2,⋯a100a_1,a_2,\cdots a_{100}a1​,a2​,⋯a100​，描述每次玩牌手中各种牌的数量。aia_iai​表示数字为iii的牌有aia_iai​张。（0≤ai≤1000\le a_i\le 1000≤ai​≤100）

Output

输出NNN行，若胡了则输出YesYesYes，否则输出NoNoNo，注意区分YesYesYes、NoNoNo的大小写！

Solution

一道比较常规的DP\mathrm{DP}DP。
记得另外一道省选麻将题可以贪心做，但此题不行，没有贪心策略。
考虑动态规划。发现一个数的牌数只可能影响它前后三个连续数的牌数（毕竟顺子只能三个连续），设f[i][j][k][0/1]f[i][j][k][0/1]f[i][j][k][0/1]表示把前iii中数字取完，取完之前数字为i−1i-1i−1的共jjj张，为iii的共kkk张，0/10/10/1表示是否取了对子，能否有取法。那么根据不同情况，可以从取二对子、三对子、四对子、顺子的情况转移过来。
除了方程特判有点多，还是挺好写的。
