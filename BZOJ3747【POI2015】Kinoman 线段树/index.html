BZOJ3747【POI2015】Kinoman <线段树>

Posted on 2018-06-25 Symbols count in article: 910 Reading time ≈ 3 mins.

Problem

【POI2015】Kinoman

Time  Limit:  60  Sec\mathrm{Time\;Limit:\;60\;Sec}TimeLimit:60Sec
Memory  Limit:  128  MB\mathrm{Memory\;Limit:\;128\;MB}MemoryLimit:128MB

Description

共有mmm部电影，编号为1∼m1\sim m1∼m，第iii部电影的好看值为w[i]w[i]w[i]。
在nnn天之中（从1∼n1\sim n1∼n编号）每天会放映一部电影，第iii天放映的是第f[i]f[i]f[i]部。
你可以选择l,r  (1≤l≤r≤n)l,r\;(1\le l\le r\le n)l,r(1≤l≤r≤n)，并观看第[l,r][l,r][l,r]天内所有的电影。
如果同一部电影你观看多于一次，你会感到无聊，于是无法获得这部电影的好看值。
你希望最大化观看且仅观看过一次的电影的好看值的总和。

Input

第一行两个整数n,m  (1≤m≤n≤106)n,m\;(1\le m\le n\le 10^6)n,m(1≤m≤n≤106)。
第二行包含nnn个整数f[1],f[2],…,f[n]  (1≤f[i]≤m)f[1],f[2],…,f[n]\;(1\le f[i]\le m)f[1],f[2],…,f[n](1≤f[i]≤m)。
第三行包含mmm个整数w[1],w[2],⋯ ,w[m]  (1≤w[j]≤106)w[1],w[2],\cdots,w[m]\;(1\le w[j]\le10^6)w[1],w[2],⋯,w[m](1≤w[j]≤106)。

Output

输出观看且仅观看过一次的电影的好看值的总和的最大值。

Sample Output

Explanation

观看第2,3,4,5,6,72,3,4,5,6,72,3,4,5,6,7天内放映的电影，其中看且仅看过一次的电影的编号为2,3,42,3,42,3,4。

Source

标签：线段树

Solution

暴力枚举左端点，考虑用线段树动态维护此时每个点作为右端点的答案的最大值。
先对每天预处理后面最早哪一天会放同样的电影，若后面不会再放，则令其为n+1n+1n+1。这个数组称为nxtnxtnxt。每种电影最先放映的时间称为lstlstlst（这是由于从后往前预处理，最先放映的在最后扫到）。
开始时，以111作为左端点，对于电影iii，右端点在[lsti,nxtlsti)[lst_i,nxt_{lst_i})[lsti​,nxtlsti​​)时iii对答案有贡献，于是区间[lsti,nxtlsti)[lst_i,nxt_{lst_i})[lsti​,nxtlsti​​)整体加wiw_iwi​。此后每次将左端点向后移一天。设当前为第iii天，则移到i+1i+1i+1后，区间[i,nxti)[i,nxt_i)[i,nxti​)不会再有电影fif_ifi​的贡献，于是区间[i,nxti)[i,nxt_i)[i,nxti​)整体减wfiw_{f_i}wfi​​。而区间[nxti,nxtnxti)[nxt_i,nxt_{nxt_i})[nxti​,nxtnxti​​)则会有电影fif_ifi​的贡献，于是区间[nxti,nxtnxti][nxt_i,nxt_{nxt_i}][nxti​,nxtnxti​​]区间加wfiw_{f_i}wfi​​。
每次移动左端点，并在线段树上修改，到下一个左端点后得到一个答案并和最大值打擂即可。时间复杂度O(nlog⁡n)O(n\log{n})O(nlogn)。
