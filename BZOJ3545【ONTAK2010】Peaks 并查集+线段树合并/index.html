BZOJ3545【ONTAK2010】Peaks <并查集+线段树合并>

Posted on 2018-06-26 Symbols count in article: 996 Reading time ≈ 4 mins.

Problem

【ONTAK2010】Peaks

Time  Limit:  10  Sec\mathrm{Time\;Limit:\;10\;Sec}TimeLimit:10Sec
Memory  Limit:  128  MB\mathrm{Memory\;Limit:\;128\;MB}MemoryLimit:128MB

Description

在Bytemountains\mathrm{Bytemountains}Bytemountains有NNN座山峰，每座山峰有他的高度hih_ihi​。
有些山峰之间有双向道路相连，共MMM条路径，每条路径有一个困难值，这个值越大表示越难走。
现在有QQQ组询问，每组询问询问从点vvv开始只经过困难值小于等于xxx的路径所能到达的山峰中第kkk高的山峰，如果无解输出−1-1−1。

Input

第一行三个数N,M,QN,M,QN,M,Q。
第二行NNN个数，第iii个数为hih_ihi​。
接下来MMM行，每行333个数a  b  ca\;b\;cabc，表示从aaa到bbb有一条困难值为ccc的双向路径。
接下来QQQ行，每行三个数v  x  kv\;x\;kvxk，表示一组询问。

Output

对于每组询问，输出一个整数表示答案。

Sample Input

Sample Output

HINT

N≤105N\le10^5N≤105，M,Q≤5×105M,Q\le5\times10^5M,Q≤5×105，hi,c,x≤109h_i,c,x\le10^9hi​,c,x≤109

Source

By Sbullet

标签：并查集 线段树合并

Solution

离线询问，按xxx从小到大排序，发现是裸的线段树合并。
将MMM条边按照边权从小到大排序，按xxx从小到大处理询问，每次只需要加边而不需要删边。这时就和BZOJ2733【HNOI2012】永无乡一样了。用并查集维护联通关系，每个连通块内用权值线段树维护。若某条边使得两个块不联通的块联通，则合并连个块的线段树，查询则直接在块内查询即可。

其实可以在线做，详见BZOJ3551【ONTAK2010】Peaks加强版。
