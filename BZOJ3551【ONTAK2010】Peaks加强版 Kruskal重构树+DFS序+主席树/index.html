BZOJ3551【ONTAK2010】Peaks加强版 < Kruskal重构树+DFS序+主席树 >

Problem

【ONTAK2010】Peaks加强版

Time  Limit:  20  Sec\mathrm{Time\;Limit:\;20\;Sec}TimeLimit:20Sec
Memory  Limit:  128  MB\mathrm{Memory\;Limit:\;128\;MB}MemoryLimit:128MB

Description

在Bytemountains\mathrm{Bytemountains}Bytemountains有NNN座山峰，每座山峰有他的高度hih_ihi​。
有些山峰之间有双向道路相连，共MMM条路径，每条路径有一个困难值，这个值越大表示越难走。
现在有QQQ组询问，每组询问询问从点vvv开始只经过困难值小于等于xxx的路径所能到达的山峰中第kkk高的山峰，如果无解输出−1-1−1。

Input

第一行三个数N,M,QN,M,QN,M,Q。
第二行NNN个数，第iii个数为hih_ihi​。
接下来MMM行，每行333个数a  b  ca\;b\;cabc，表示从aaa到bbb有一条困难值为ccc的双向路径。
接下来QQQ行，每行三个数v  x  kv\;x\;kvxk，表示一组询问。
v=v  xor  lastansv=v\;\mathrm{xor}\;lastansv=vxorlastans，x=x  xor  lastansx=x\;\mathrm{xor}\;lastansx=xxorlastans，k=k  xor  lastansk=k\;\mathrm{xor}\;lastansk=kxorlastans，如果lastans=−1lastans=-1lastans=−1则不变。

Output

对于每组询问，输出一个整数表示答案。

Solution

这道题是BZOJ3545【ONTAK2010】Peaks的强制在线版本。
无法离线，则不能只加边不删边，于是无法直接线段树合并。

这里需要引入Kruskal重构树\mathrm{Kruskal重构树}Kruskal重构树这一概念。
对于一个图，我们对其构建Kruskal重构树\mathrm{Kruskal}重构树Kruskal重构树：
为每条边都设置一个点，点权为该边边权；为每个点（连同边代表的点）一起建立并查集。

  1. 找出当前没有尝试过的最小的边iii，判断其两端点uiu_iui​和viv_ivi​的连通性
  2. 若两端点尚未联通，则将uiu_iui​和viv_ivi​的祖先用并查集并到边iii所代表的点pip_ipi​上，这时在重构树上加边pi→uip_i\to u_ipi​→ui​,pi→vip_i\to v_ipi​→vi​
  3. 若当前所有原树上的点都在同一个并查集中，则退出，否则返回步骤111

下图是一个Kruskal重构树\mathrm{Kruskal}重构树Kruskal重构树的例子，其四个性质在下方标注

不难发现，由于其是一个大根堆，从一个点出发，只走边权小于等于xxx的点所能到达的点，一定是重构树上一个边所代表的点的子树的所有叶子节点。这样我们在重构树上倍增预处理一下，即可在log⁡n\log{n}logn的时间内找到每个询问对应的上述的这个点。对整棵重构树做树上主席树，即可在log⁡n\log{n}logn的时间内找到第kkk大的点权。

综上，先跑Kruskal\mathrm{Kruskal}Kruskal建立重构树，并在重构树上倍增预处理和树上值域主席树预处理。随后对每次询问，先倍增跳到权值小于等于xxx的层数最浅的结点，再在此结点子树的DFS\mathrm{DFS}DFS序区间中询问第kkk大权值即可，时间复杂度O(nlog⁡n)O(n\log{n})O(nlogn)。
