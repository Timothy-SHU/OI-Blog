Problem

【CERC2017】Gambling Guide

Time  Limit:  20  Sec\mathrm{Time\;Limit:\;20\;Sec}TimeLimit:20Sec
Memory  Limit:  512  MB\mathrm{Memory\;Limit:\;512\;MB}MemoryLimit:512MB
Special  Judge\mathrm{Special\;Judge}SpecialJudge

Description

给定一张nnn个点，mmm条双向边的无向图。
你要从111号点走到nnn号点。当你位于xxx点时，你需要花111元钱，等概率随机地买到与xxx相邻的一个点的票，只有通过票才能走到其它点。
每当完成一次交易时，你可以选择直接使用那张票，也可以选择扔掉那张票然后再花111元钱随机买另一张票。注意你可以无限次扔票。
请使用最佳的策略，使得期望花的钱数最少。

Input

第一行包含两个正整数n,mn,mn,m (1≤n,m≤3×105)(1\le n,m\le3\times10^5)(1≤n,m≤3×105)，表示点数和边数。
接下来mmm行，每行两个正整数u,vu,vu,v (1≤u,v≤n)(1\le u,v\le n)(1≤u,v≤n)，表示一条双向边。
输入数据保证无重边、无自环，且111号点一定可以走到nnn号点。

Output

输出一行一个实数，即最少的期望花费，当绝对或者相对误差不超过10−610^{-6}10−6时视为正确。

Solution

好题，精妙的DP\mathrm{DP}DP转移。

原题意相当于从111走到nnn，每步可以随机一个相邻的点，选择走或不走，求期望步数。

令f(x)f(x)f(x)为从xxx到nnn的期望步数。如果随出来的相邻点为yyy，且f(y)>f(x)f(y)>f(x)f(y)>f(x)，那么肯定不走最优；否则一定走到yyy更优。令xxx的出边走向的点集为TTT，有f(x)=∑y∈Tmin⁡(f(x),f(y))deg(x)f(x)=\sum_{y\in T}\frac{\min(f(x),f(y))}{deg(x)}f(x)=∑y∈T​deg(x)min(f(x),f(y))​。

考虑以何种顺序进行这个DP\mathrm{DP}DP。最开始时，只有f(n)=0f(n)=0f(n)=0，这个值是确定的，于是我们将nnn加入到一个集合{S}\lbrace S\rbrace{S}中。集合{S}\lbrace S\rbrace{S}为现在“固定(fixed)”了的点的集合，在SSS中的点的fff值一定比没在其中的点的fff值小。考虑用SSS中固定了的fff值去更新相邻的点，对于这些未被“固定”的相邻的点，如果其走向已经被”固定“的点，那么一定更优；否则，一定没有留在当前点优。
那么有

f(x)=1+∑neighbour  y∈Sf(y)deg(x)+∑neighbour  y∉Sf(x)deg(x)f(x)=1+\sum_{\mathrm{neighbour}\;y\in S}\frac{f(y)}{deg(x)}+\sum_{\mathrm{neighbour}\;y\notin S}\frac{f(x)}{deg(x)} f(x)=1+neighboury∈S∑​deg(x)f(y)​+neighboury∈/S∑​deg(x)f(x)​

移项化简得

f(x)=deg(x)+∑neighbour  y∈Sf(y)deg(x)−∑neighbour  y∉S1f(x)=\frac{deg(x)+\sum_{\mathrm{neighbour}\;y\in S}f(y)}{deg(x)-\sum_{\mathrm{neighbour}\;y\notin S}1} f(x)=deg(x)−∑neighboury∈/S​1deg(x)+∑neighboury∈S​f(y)​

由于在SSS中的点fff值一定比外面的小，已经被”固定“的点不可能因为走向没被”固定“的点而变得更优，所以一个点一旦固定了，它的fff值一定固定了，我们用它去更新其他的点。每次操作时，我们选出没被固定的点中fff值最小的点（因为这个值不可能再被其他没被固定的点更新了），然后用它更新周围的点。

我们发现这个过程和Dijkstra\mathrm{Dijkstra}Dijkstra单源最短路极为相似，都是每次固定一个距离最小的点，更新其周围的点。和Dijkstra\mathrm{Dijkstra}Dijkstra一样，我们可以用堆来优化找fff最小的未被固定的点的过程。这样总时间复杂度O((N+M)log⁡N)O((N+M)\log{N})O((N+M)logN)。
