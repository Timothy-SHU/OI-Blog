BZOJ1178【APIO2009】会议中心 <贪心+倍增>

Posted on 2018-04-20 Symbols count in article: 1.5k Reading time ≈ 5 mins.

Problem

【APIO2009】会议中心

Time  Limit:  15  Sec\mathrm{Time\;Limit:\;15\;Sec}TimeLimit:15Sec
Memory  Limit:  162  MB\mathrm{Memory\;Limit:\;162\;MB}MemoryLimit:162MB

Description

Siruseri\mathrm{Siruseri}Siruseri政府建造了一座新的会议中心。许多公司对租借会议中心的会堂很感兴趣，他们希望能够在里面举行会议。
对于一个客户而言，仅当在开会时能够独自占用整个会堂，他才会租借会堂。会议中心的销售主管认为：最好的策略应该是将会堂租借给尽可能多的客户。
显然，有可能存在不止一种满足要求的策略。 例如下面的例子。总共有444个公司。他们对租借会堂发出了请求，并提出了他们所需占用会堂的起止日期（如下表所示）。

公司 开始日期 结束日期
公司1公司1公司1 444 999
公司2公司2公司2 999 111111
公司3公司3公司3 131313 191919
公司4公司4公司4 101010 171717

上例中，最多将会堂租借给两家公司。租借策略分别是租给公司1公司1公司1和公司3公司3公司3， 或是公司2公司2公司2和公司3公司3公司3，也可以是公司1公司1公司1和公司4公司4公司4。注意会议中心一天最多租借给 一个公司，所以公司1公司1公司1和公司2公司2公司2不能同时租借会议中心，因为他们在第九天重合 了。
销售主管为了公平起见，决定按照如下的程序来确定选择何种租借策略：首先，将租借给客户数量最多的策略作为候选，将所有的公司按照他们发出请求的 顺序编号。对于候选策略，将策略中的每家公司的编号按升序排列。最后，选出其中字典序最小的候选策略作为最终的策略。
例中，会堂最终将被租借给公司1公司1公司1和公司3公司3公司3：333个候选策略是 {(1,3),(2,3),(1,4)}\lbrace(1,3),(2,3),(1,4)\rbrace{(1,3),(2,3),(1,4)}，而在字典序中(1,3)<(1,4)<(2,3)(1,3)<(1,4)<(2,3)(1,3)<(1,4)<(2,3)。
你的任务是帮助销售主管确定应该将会堂租借给哪些公司。

对于每个公司的申请，起始日期为不小于111的整数，终止日期为不大于10910^9109的整数。

第二行应列出MMM个数，表示最终将会堂租借给哪些公司。

Solution

思路清奇的贪心…

如果没有“字典序最小”，直接无脑贪心即可。有输方案的要求后，就需要用另外一种贪心。

首先先做一遍贪心找到最多能有多少个会议，然后从111到nnn按编号枚举会议，看当前会议加入后会不会使答案变小，如果不会变小就贪心把这个会议加入到方案中。

具体地，首先以rrr从小到大为第一关键字，以lll从大到小为第二关键字排序，去掉包含的区间。
找到一种方法（一会儿说）计算F(p,q)F(p,q)F(p,q)，表示p∼qp\sim qp∼q时间段最多可以放多少个会议。
对于会议iii，其时间段是[l,r][l,r][l,r]，已经加入的会议中此会议的前驱的结束时间是lrlrlr，后继的开始时间是rlrlrl（前驱和后继用set维护）。

  • 若l≤lrl\le lrl≤lr或r≥rlr\ge rlr≥rl，那么一定不能放入。
  • 若F(lr+1,l−1)+F(r+1,rl−1)+1=F(lr+1,rl−1)F(lr+1,l-1)+F(r+1,rl-1)+1=F(lr+1,rl-1)F(lr+1,l−1)+F(r+1,rl−1)+1=F(lr+1,rl−1)，那么放入后一定不会影响答案，输出编号后把此会议加入set即可。这个等式表示加入此会议后虽然把原区间分成了三个子区间，但最大值依旧不变。
  • 若F(lr+1,l−1)+F(r+1,rl−1)+1≠F(lr+1,rl−1)F(lr+1,l-1)+F(r+1,rl-1)+1\ne F(lr+1,rl-1)F(lr+1,l−1)+F(r+1,rl−1)+1=F(lr+1,rl−1)，那么不能放入。

这样一来，就可以解决输出方案的问题了。

但如何计算F(p,q)F(p,q)F(p,q)呢？
如果直接算，是O(n)O(n)O(n)的，考虑把这个过程变成O(log⁡n)O(\log n)O(logn)。
倍增预处理出nxt[u][i]nxt[u][i]nxt[u][i]，表示从区间uuu开始向后选2i2^i2i个连续区间，最后一个区间的编号。于是每次计算可以倍增跳累加答案。
