BZOJ4205【FJ2015集训】卡牌配对 <网络流>

Posted on 2017-12-25 Symbols count in article: 1.2k Reading time ≈ 4 mins.

Problem

【FJ2015集训】卡牌配对

Time  Limit:  20  Sec\mathrm{Time\;Limit:\;20\;Sec}TimeLimit:20Sec
Memory  Limit:  512  MB\mathrm{Memory\;Limit:\;512\;MB}MemoryLimit:512MB

Description

现在有一种卡牌游戏，每张卡牌上有三个属性值：A,B,CA,B,CA,B,C。把卡牌分为X,YX,YX,Y两类，分别有n1,n2n_1,n_2n1​,n2​张。
两张卡牌能够配对，当且仅当，存在至多一项属性值使得两张卡牌该项属性值互质，且两张卡牌类别不同。
比如一张XXX类卡牌属性值分别是225,233,101225,233,101225,233,101，一张YYY类卡牌属性值分别为115,466,99115,466,99115,466,99。那么这两张牌是可以配对的，因为只有101101101和999999一组属性互质。
游戏的目的是最大化匹配上的卡牌组数，当然每张卡牌只能用一次。

Input

数据第一行两个数n1n_1n1​，n2n_2n2​，空格分割。
接下来n1n_1n1​行，每行333个数，依次表示每张XXX类卡牌的333项属性值。
接下来n2n_2n2​行，每行333个数，依次表示每张YYY类卡牌的333项属性值。

Output

输出一个整数：最多能够匹配的数目。

HINT

样例中第一张XXX类卡牌和第一张YYY类卡牌能配对，第二张XXX类卡牌和两张YYY类卡牌都能配对。所以最佳方案是第一张XXX和第一张YYY配对，第二张XXX和第二张YYY配对。
对于100%100\%100%的数据，n1,n2≤30000n_1,n_2\le 30000n1​,n2​≤30000，属性值为不超过200200200的正整数
请大胆使用渐进复杂度较高的算法！

标签：网络流

Solution

非常巧妙的一道二分图建模
首先考虑暴力建边，显然边数是 n2n^2n2 级别的，会同时MLEMLEMLE和TLETLETLE
注意到值域只有200200200，可以预处理出所有质数，并且可以发现可匹配的情况只有三种，即(A,B)(A,B)(A,B)不互质,(B,C)(B,C)(B,C)不互质,(C,A)(C,A)(C,A)不互质（这里不互质只两张卡的两种同样属性不互质）。这样总共的不互质即可以匹配的情况共有46×46×346\times 46\times 346×46×3种（200200200内共464646个质数）
对于一张XXX类卡牌(A,B,C)(A,B,C)(A,B,C)，如果一个点(a,b)(a,b)(a,b)，aaa是AAA的质因子，bbb是BBB的质因子，那么就连一条由卡牌出发到那个点的流量为111的边。其他类的类似。YYY类卡牌的话就连一条点到卡牌的边。源点向每个XXX牌连一条流量为111的边，每个YYY牌向汇点连一条流量为111的边。构图后跑最大流即可。
