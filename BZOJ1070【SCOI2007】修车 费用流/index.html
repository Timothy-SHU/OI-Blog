BZOJ1070【SCOI2007】修车 <费用流>

Posted on 2017-10-17 Symbols count in article: 751 Reading time ≈ 3 mins.

Problem

【SCOI2007】修车

Time  Limit:  1  Sec\mathrm{Time\;Limit:\;1\;Sec}TimeLimit:1Sec
Memory  Limit:  128  MB\mathrm{Memory\;Limit:\;128\;MB}MemoryLimit:128MB

Description

同一时刻有NNN位车主带着他们的爱车来到了汽车维修中心。维修中心共有MMM位技术人员，不同的技术人员对不同的车进行维修所用的时间是不同的。现在需要安排这MMM位技术人员所维修的车及顺序，使得顾客平均等待的时间最小。
说明：顾客的等待时间是指从他把车送至维修中心到维修完毕所用的时间。

Input

第一行有两个m,nm,nm,n，表示技术人员数与顾客数。 接下来nnn行，每行mmm个整数。第i+1i+1i+1行第jjj个数表示第jjj位技术人
员维修第iii辆车需要用的时间TTT。

Output

最小平均等待时间，答案精确到小数点后222位。

Sample Input

Sample Output

HINT

数据范围: 2≤M≤92\le M\le 92≤M≤9, 1≤N≤601\le N\le 601≤N≤60, 1≤T≤10001\le T\le 10001≤T≤1000

标签：费用流

Solution

此题建模是费用流中常见的拆点建模。
把每个技术人员拆成nnn个点，第kkk个点代表着此工作人员修的倒数第kkk辆车对答案的贡献。
对于每个顾客，如果他倒数第kkk个修车，花费ttt的时间，会对答案产生t×kt\times kt×k的贡献。
因此从每个顾客向每个技术人员的nnn个点连边，容量为111，花费为t×kt\times kt×k，然后跑费用流即可。
