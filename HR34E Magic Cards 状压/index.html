HR34E Magic Cards <状压>

Posted on 2018-09-19 Symbols count in article: 1.4k Reading time ≈ 5 mins.

Problem

Magic Cards

by Birjik

Description

Birzhan has nnn cards, numbered from 111 to nnn. Every card has each number from 111 to mmm written on it. Some numbers exist on the front side of the card and some on the back side. No number exists on both the sides of a card at the same time. These cards are placed on the table in a row such that only one side is visible. Birzhan is allowed to flip them any number of times.
Now, Birzhan has to answer qqq queries each of them consisting of two integers lll and rrr (1≤l≤r≤n1\le l\le r\le n1≤l≤r≤n). We define f(l,r)f(l,r)f(l,r) as the sum of the squares of every integer from 111 to mmm if it exists on the visible side of any card numbered from lll to rrr. Given that Birzhan can flip any number of cards any number of times, find and print the maximum value of f(l,r)f(l,r)f(l,r).
For example, given 333 cards and m=5m=5m=5 we have the as follows:-

Now, for l=1l=1l=1 and r=2r=2r=2. We have 1,2,4,51,2,4,51,2,4,5 on the two cards, the sum of their squares is 464646.
If we flip the first card, we get 3,4,53,4,53,4,5, the sum of their squares is 505050.
And, if we flip both cards, we get 1,2,3,4,51,2,3,4,51,2,3,4,5, the sum of their squares is 555555
Hence, the maximum value of f(1,2)f(1,2)f(1,2) in the above case is 555555.

Input Format

The first line contains three space-separated integers, nnn, mmm, and qqq, denoting the number of cards, what numbers written on each card, and the number of queries, respectively.
Each of the next nnn lines describes the cards. On each line, the first number is xix_ixi​, denoting the count of numbers written on the visible side of the ithi^{th}ith card. Next xix_ixi​ space-separated unique integers represent the numbers written on the visible side of that card, each between 111 and mmm.
Next qqq lines contain two space-separated integers, lll and rrr, describing the query mentioned above.

Constraints

  • 1≤n×m,q≤1061\le n\times m,q\le 10^61≤n×m,q≤106
  • 0≤xi≤m0\le x_i\le m0≤xi​≤m
  • 1≤l≤r≤n1\le l\le r\le n1≤l≤r≤n

Output Format

Print the maximum value of f(l,r)f(l,r)f(l,r) for each query.

Translation

题目大意：有nnn张牌，每张牌都写上了111到mmm的所有正整数各一次，其中有的数字写在牌的正面，有的写在反面。有qqq次询问，每次询问一个区间[l,r][l,r][l,r]的所有牌，每张牌可以任意决定正面朝上还是反面朝上，问朝上的牌中所有出现过至少一次的数的平方和的最大能是多少。

Solution

没结论是神仙题，有结论立马变水题。

结论：若牌的数量大于⌈log⁡2m⌉\lceil\log_2m\rceil⌈log2​m⌉，则一定可以通过某种方式使得所有数都至少出现一次。
证明：对于第iii张牌，若前面还没出现过的数有kkk个，那么这张牌取正面或反面至少可以出现⌈k2⌉\lceil\frac{k}{2}\rceil⌈2k​⌉个前面未出现过的数。于是⌈log⁡2m⌉\lceil\log_2m\rceil⌈log2​m⌉张牌可以保证出现所有mmm个数。

对于询问[l,r][l,r][l,r]，若r−l+1>log⁡2mr-l+1>\log_2mr−l+1>log2​m，则答案为sum=12+22+⋯+m2sum=1^2+2^2+\cdots+m^2sum=12+22+⋯+m2，否则，区间长度一定不大于202020。预处理出长度不大于202020的所有区间的最大可能答案，直接回答询问即可。

接下来用状态压缩加速预处理。对于区间[l,r][l,r][l,r]的一种方案，用r−l+1r-l+1r−l+1位二进制数表示，其中000表示这一位置的牌背面朝上，111表示这一位置的牌正面朝上。对于1∼m1\sim m1∼m中的每一个数，均可以处理出一个数SSS，其中第iii位表示第iii个位置的牌上这个数是在正面(111)还是背面(000)。那么当且仅当这个区间的方案为¬S\neg S¬S时，才不可能出现这个数。用sis_isi​表示情况iii下所有不能出现的数的平方和。处理每个数对应的SSS并在s¬Ss_{\neg S}s¬S​减去这个数的平方，然后枚举每种方案取最大值即可。用滑动窗口即可在O(m)O(m)O(m)的时间内从一个区间转移到另一个区间，如此即可在O(nmlog⁡m)O(nm\log m)O(nmlogm)的复杂度下进行预处理。
