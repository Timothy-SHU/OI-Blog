BZOJ3730 震波 <动态点分治>

Posted on 2018-04-10 Symbols count in article: 1.4k Reading time ≈ 5 mins.

Problem

震波

Time  Limit:  15  Sec\mathrm{Time\;Limit:\;15\;Sec}TimeLimit:15Sec
Memory  Limit:  256  MB\mathrm{Memory\;Limit:\;256\;MB}MemoryLimit:256MB

Description

在一片土地上有NNN个城市，通过N−1N-1N−1条无向边互相连接，形成一棵树的结构，相邻两个城市的距离为111，其中第i个城市的价值为valueivalue_ivaluei​。
不幸的是，这片土地常常发生地震，并且随着时代的发展，城市的价值也往往会发生变动。
接下来你需要在线处理MMM次操作：

  • 0  x  k0\;x\;k0xk 表示发生了一次地震，震中城市为xxx，影响范围为kkk，所有与xxx距离不超过kkk的城市都将受到影响，该次地震造成的经济损失为所有受影响城市的价值和。
  • 1  x  y1\;x\;y1xy 表示第xxx个城市的价值变成了yyy。

为了体现程序的在线性，操作中的x,y,kx,y,kx,y,k都需要异或你程序上一次的输出来解密，如果之前没有输出，则默认上一次的输出为000。

Input

第一行包含两个正整数NNN和MMM。
第二行包含NNN个正整数，第iii个数表示valueivalue_ivaluei​。
接下来N−1N-1N−1行，每行包含两个正整数u,vu,vu,v，表示uuu和vvv之间有一条无向边。
接下来MMM行，每行包含三个数，表示MMM次操作。

Output

包含若干行，对于每个询问输出一行一个正整数表示答案。

标签：动态点分治

Solution

终于学会动态点分辣QAQ~
赶快来肝几道基础题

先不考虑时间复杂度，那么对于每个000操作显然可以暴力从震源向上爬统计答案。每次加上当前子树中距离符合题意的点，再减去其儿子（就是向上爬时此点的前驱）的子树中与其算重复的点。这样维护两个树状数组即可。
但是树高可以构造比log⁡n\log nlogn大，这时就需要建立点分树，把树高降成log⁡n\log nlogn。

提取重心建立点分树，对每个分治中心维护两个树状数组，第一个是其子树中到此分治中心的每种距离的所有点的点权和，第二个是其子树中到此分治中心的上一层分治中心的每种距离的所有点的点权和。
这样对于询问，每次log⁡n\log nlogn向上爬，爬到每个分治中心log⁡size\log sizelogsize统计；对于修改，每次log⁡n\log nlogn向上爬，爬到每个分治中心log⁡size\log sizelogsize更新树状数组即可。这样总复杂度是O(Qlog⁡2n)O(Q\log^2n)O(Qlog2n)。

不过此题有些卡常，有三种策略：

  1. 用RMQ\mathrm{RMQ}RMQ做LCA\mathrm{LCA}LCA
  2. 带fread大读优
  3. 将nnn棵树状数组建到同一个大数组上，每个BIT\mathrm{BIT}BIT记录其起始指针和终止指针，可以避免vector的一些常数
