BZOJ3442 学习小组 <费用流>

Posted on 2017-12-27 Symbols count in article: 1.1k Reading time ≈ 4 mins.

Problem

学习小组

Time Limit: 5Sec5 Sec5Sec
Memory Limit: 128MB128 MB128MB

Description

坑校准备鼓励学生参加学习小组。
共有nnn个学生，mmm个学习小组，每个学生有一定的喜好，只愿意参加其中的一些学习小组，但是校领导为学生考虑，规定一个学生最多参加kkk个学习小组。财务处的大叔就没那么好了，他想尽量多收钱，因为每个学生参加学习小组都要交一定的手续费，不同的学习小组有不同的手续费。然而，事与愿违，校领导又决定对学习小组组织者进行奖励，若有aaa个学生参加第i个学习小组，那么给这个学习小组组织者奖励Ci×a2C_i\times a^2Ci​×a2元。在参与学生（而不是每个学习小组的人数总和）尽量多的情况下，求财务处最少要支出多少钱（若为负数，则输出负数）（支出=总奖励费−总手续费支出=总奖励费-总手续费支出=总奖励费−总手续费）。

Input

输入有若干行，第一行有三个用空格隔开的正整数n、m、kn、m、kn、m、k。接下来的一行有mmm个正整数，表示每个CiC_iCi​。第三行有mmm个正整数，表示参加每个学习小组需要交的手续费FiF_iFi​。再接下来有一个nnn行mmm列的矩阵，表若第iii行jjj列的数字是111，则表示第iii个学生愿意参加第jjj个学习小组，若为000，则为不愿意。

Output

输出只有一个整数，为最小的支出。

Hint

样例解释
参与学生最多为333，每个学生参加一个学习小组，若有两个学生参加第一个学习小组，一个学生参加第二个学习小组（一定要有人参加第二个学习小组），支出为−2-2−2，可以证明没有更优的方案了。
数据范围与约定
100%100\%100%的数据，0＜n≤100，0＜m≤90，0＜k≤m，0＜Ci≤10，0＜Fi≤1000＜n\le 100，0＜m\le 90，0＜k\le m，0＜Ci\le 10，0＜Fi\le 1000＜n≤100，0＜m≤90，0＜k≤m，0＜Ci≤10，0＜Fi≤100。

标签：费用流

Solution

常规费用流建模。
建图：
每个学生为一个点，每个小组为一个点，共n+mn+mn+m个点。
对每个学生pip_ipi​，
S→piS\to p_iS→pi​ 流量kkk 费用000 （限制最多选kkk个组）
pi→Tp_i\to Tpi​→T 流量k−1k-1k−1 费用000 （限制至少选一个组）
对每个组gig_igi​，
gi→Tg_i\to Tgi​→T 流量111 费用Ci×(2j−1)C_i\times (2j-1)Ci​×(2j−1) 其中j∈[1,n]j\in [1, n]j∈[1,n]
即当前若有xxx个人，再多一个人会带来Ci×(x+1)2−Ci×x2=Ci×(2x+1)=Ci×[2(x+1)−1]C_i\times (x+1)^2-C_i\times x^2=C_i\times (2x+1)=C_i\times [2(x+1)-1]Ci​×(x+1)2−Ci​×x2=Ci​×(2x+1)=Ci​×[2(x+1)−1]的收益
学生和组之间则连边 pi→gjp_i\to g_jpi​→gj​ 流量111 费用−Fi-F_i−Fi​
跑小费流即可。
