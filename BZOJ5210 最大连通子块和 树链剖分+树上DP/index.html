BZOJ5210 最大连通子块和 <树链剖分+树形DP>

Posted on 2018-06-05 Symbols count in article: 1.5k Reading time ≈ 5 mins.

Problem

最大连通子块和

Time  Limit:  20  Sec\mathrm{Time\;Limit:\;20\;Sec}TimeLimit:20Sec
Memory  Limit:  128  MB\mathrm{Memory\;Limit:\;128\;MB}MemoryLimit:128MB

Description

给出一棵nnn个点，以111为根的有根树，点有点权。
要求支持如下两种操作：

  1. M  x  y\mathrm{M}\;x\;yMxy：将点xxx的点权改为yyy
  2. Q  x\mathrm{Q}\;xQx：求以xxx为根的子树的最大连通子块和

一棵子树的最大连通子块和指该子树所有子连通块的点权和中的最大值（本题中子连通块包括空连通块，点权和为000）。

Input

第一行两个整数nnn,mmm，表示树的点数以及操作的数目。
第二行nnn个整数，第iii个整数wiw_iwi​表示第iii个点的点权。
接下来的n−1n-1n−1行，每行两个整数x,yx,yx,y，表示xxx和yyy之间有一条边相连。
接下来的mmm行，每行输入一个操作，含义如题目所述。
保证操作为M  x  y\mathrm{M}\;x\;yMxy或Q  x\mathrm{Q}\;xQx之一。

Output

对于每个Q\mathrm{Q}Q操作输出一行一个整数，表示询问子树的最大连通子块和。

Solution

经典树链剖分维护树上DP\mathrm{DP}DP。以下解法源自出题人CQzhangyu的博客和GXZlegend的博客。

首先考虑暴力DP\mathrm{DP}DP，令f[x]f[x]f[x]表示xxx子树中包含xxx的连通块权值和最大值，那么f[x]=max⁡(0,w[x]+∑x→y∈Gf[y])f[x]=\max(0, w[x]+\sum_{x\to y\in\mathrm{G}}f[y])f[x]=max(0,w[x]+∑x→y∈G​f[y])。维护s[x]s[x]s[x]表示xxx子树中连通块权值和最大值，则s[x]=max⁡(f[x],max⁡x→y∈Gs[y])s[x]=\max(f[x],\max_{x\to y\in\mathrm{G}}s[y])s[x]=max(f[x],maxx→y∈G​s[y])。每次修改后重新DP\mathrm{DP}DP，可做到O(n2)O(n^2)O(n2)。

注意到每次修改后不是所有的fff都变化。用树链剖分维护树上DP\mathrm{DP}DP，可以每次不修改所有的fff值。然而直接维护fff的值不方便，因为递推式中的和式在线段树上不便于计算。于是引入g[x]=w[x]+∑f[y]g[x]=w[x]+\sum f[y]g[x]=w[x]+∑f[y]，其中yyy是xxx的轻儿子。那么重链上的转移就变为f[x]=max⁡(0,f[son[x]]+g[x])f[x]=\max(0,f[son[x]]+g[x])f[x]=max(0,f[son[x]]+g[x])。这其实就是最大连续子段和的DP\mathrm{DP}DP方式，可以用线段树维护带修改最大连续子段和。于是对于每次修改，向上跳重链，在线段树上每条重链的区域内维护最大连续子段和即可。注意这里“每条重链的区域”指的是链顶到链底的距离，而非括号序列。

再考虑如何维护sss。注意到sss每次都直接取最值，带修改后，其实是可删除堆的形式。对每个点维护可删除堆来维护轻儿子的sss值最大值，对于重儿子则在线段树上维护。在线段树上修改后update\mathrm{update}update时，用每个结点的sss堆顶元素更新最大子段和，即可动态维护sss。

查询时，直接向上跳重链，将该重链对应区间的最大子段和取出来打擂即可。

这样修改复杂度O(log⁡2n)O(\log^2n)O(log2n)，查询复杂度O(log⁡n)O(\log n)O(logn)，总复杂度O(mlog⁡2n)O(m\log^2n)O(mlog2n)。
