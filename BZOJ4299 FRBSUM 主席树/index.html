BZOJ4299 FRBSUM <主席树>

Posted on 2018-04-24 Symbols count in article: 751 Reading time ≈ 3 mins.

Problem

FRBSUM

Time  Limit:  10  Sec\mathrm{Time\;Limit:\;10\;Sec}TimeLimit:10Sec
Memory  Limit:  128  MB\mathrm{Memory\;Limit:\;128\;MB}MemoryLimit:128MB

Description

数集SSS的Forbidden  Sum\mathrm{Forbidden\;Sum}ForbiddenSum定义为无法用SSS的某个子集（可以为空）的和表示的最小的非负整数。
例如，S=1,1,3,7S={1,1,3,7}S=1,1,3,7，则它的子集和中包含0(S′=∅)0(S'=\emptyset)0(S′=∅)，1(S′=1)1(S'={1})1(S′=1)，2(S′=1,1)2(S'={1,1})2(S′=1,1)，3(S′=3)3(S'={3})3(S′=3)，4(S′=1,3)4(S'={1,3})4(S′=1,3)，5(S′=1,1,3)5(S'={1,1,3})5(S′=1,1,3)，但是它无法得到666。因此SSS的Forbidden  Sum\mathrm{Forbidden\;Sum}ForbiddenSum为666。
给定一个序列AAA，你的任务是回答该数列的一些子区间所形成的数集的Forbidden  Sum\mathrm{Forbidden\;Sum}ForbiddenSum是多少。

接下来MMM行，每行一对整数，表示一组询问。

Solution

本题和【FJOI2016】神秘数相同，双倍经验。

首先不难发现一个结论，若某集合当前能凑出0∼mx0\sim mx0∼mx中的所有数，加入一个数xxx，可凑出的数的值域扩展当且仅当x≤mx+1x\le mx+1x≤mx+1，并且会将值域扩展到0∼mx+x0\sim mx+x0∼mx+x。
如此，对于每个区间[l,r][l,r][l,r]，从小到大逐一将每个数加入到集合中，像上面那样不断扩展值域，如果加入某个数时x>mx+1x>mx+1x>mx+1，值域无法继续扩充，那么mx+1mx+1mx+1即为最小的不能凑成的数。
这个过程可以用一棵值域主席树维护，每次将所有小于等于mx+1mx+1mx+1的数求和，作为新的mxmxmx，若mxmxmx在某次这样的操作中不变，则无法继续扩展，输出答案mx+1mx+1mx+1。
