Problem

【NOI2004】郁闷的出纳员

Time  Limit:  5  Sec\mathrm{Time\;Limit:\;5\;Sec}TimeLimit:5Sec
Memory  Limit:  64  MB\mathrm{Memory\;Limit:\;64\;MB}MemoryLimit:64MB

Description

OIER\mathrm{OIER}OIER公司是一家大型专业化软件公司，有着数以万计的员工。作为一名出纳员，我的任务之一便是统计每位员工的工资。
这本来是一份不错的工作，但是令人郁闷的是，我们的老板反复无常，经常调整员工的工资。如果他心情好，就可能把每位员工的工资加上一个相同的量。反之，如果心情不好，就可能把他们的工资扣除一个相同的量。我真不知道除了调工资他还做什么其它事情。
工资的频繁调整很让员工反感，尤其是集体扣除工资的时候，一旦某位员工发现自己的工资已经低于了合同规定的工资下界，他就会立刻气愤地离开公司，并且再也不会回来了。每位员工的工资下界都是统一规定的。每当一个人离开公司，我就要从电脑中把他的工资档案删去，同样，每当公司招聘了一位新员工，我就得为他新建一个工资档案。
老板经常到我这边来询问工资情况，他并不问具体某位员工的工资情况，而是问现在工资第kkk多的员工拿多少工资。每当这时，我就不得不对数万个员工进行一次漫长的排序，然后告诉他答案。
好了，现在你已经对我的工作了解不少了。正如你猜的那样，我想请你编一个工资统计程序。怎么样，不是很困难吧？

Input

第一行有两个非负整数nnn和minminmin。nnn表示下面有多少条命令，minminmin表示工资下界。
接下来的nnn行，每行表示一条命令。命令可以是以下四种之一：

名称 格式 作用
I\mathrm{I}I命令 I  kI\;kIk 新建一个工资档案，初始工资为kkk
A\mathrm{A}A命令 A  kA\;kAk 把每位员工的工资加上kkk
S\mathrm{S}S命令 S  kS\;kSk 把每位员工的工资扣除kkk
F\mathrm{F}F命令 F  kF\;kFk 查询第kkk多的工资

I\mathrm{I}I命令、A\mathrm{A}A命令、S\mathrm{S}S命令中的kkk是一个非负整数，F\mathrm{F}F命令中的kkk是一个正整数。
在初始时，可以认为公司里一个员工也没有。
I\mathrm{I}I命令的条数不超过10510^5105
A\mathrm{A}A命令和S\mathrm{S}S命令的总条数不超过100100100
F\mathrm{F}F命令的条数不超过10510^5105
每次工资调整的调整量不超过100010001000
新员工的工资不超过10510^5105

Output

输出行数为F\mathrm{F}F命令的条数加一。
对于每条F\mathrm{F}F命令，你的程序要输出一行，仅包含一个整数，为当前工资第kkk多的员工所拿的工资数，
如果kkk大于目前员工的数目，则输出−1-1−1。
输出文件的最后一行包含一个整数，为离开公司的员工的总数。

Solution

Splay\mathrm{Splay}Splay基础题。

由于加和减都是作用于全局，我们可以单独记当前加和减操作造成的总偏差是多少，记为detdetdet，即当前序列中每个数的真实值为其在序列中的值加detdetdet。这样新加入一个大小为xxx的数时，相当于在序列中加入一个大小为x−detx-detx−det的数。
对于每个I\mathrm{I}I命令，在Splay\mathrm{Splay}Splay中插入一个数x−detx-detx−det。
对于每个A\mathrm{A}A命令，det=det+xdet=det+xdet=det+x。
对于每个S\mathrm{S}S命令，det=det−xdet=det-xdet=det−x，再将序列中所有小于min−detmin-detmin−det的数删除。具体地，将保护结点−∞-\infty−∞转到根，将min−det−1min-det-1min−det−1的后继结点转到根的右儿子，将根的右儿子的左儿子整体删除。
对于每个F\mathrm{F}F命令，询问第xxx小的值。
