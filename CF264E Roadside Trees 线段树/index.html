CF264E Roadside Trees <线段树>

Posted on 2018-10-01 Symbols count in article: 1.5k Reading time ≈ 6 mins.

Problem

Roadside Trees

Time  limit:  5  Sec\mathrm{Time\;limit:\;5\;Sec}Timelimit:5Sec
Memory  limit:  256  MB\mathrm{Memory\;limit:\;256\;MB}Memorylimit:256MB

Description

Squirrel Liss loves nuts. Liss asks you to plant some nut trees.
There are nnn positions (numbered 1 to nnn from west to east) to plant a tree along a street. Trees grow one meter per month. At the beginning of each month you should process one query. The query is one of the following types:

  • Plant a tree of height hhh at position ppp.
  • Cut down the xthx^{th}xth existent (not cut) tree from the west (where xxx is 1-indexed). When we cut the tree it drops down and takes all the available place at the position where it has stood. So no tree can be planted at this position anymore.

After processing each query, you should print the length of the longest increasing subsequence. A subset of existent trees is called an increasing subsequence if the height of the trees in the set is strictly increasing from west to east (for example, the westmost tree in the set must be the shortest in the set). The length of the increasing subsequence is the number of trees in it.
Note that Liss don’t like the trees with the same heights, so it is guaranteed that at any time no two trees have the exactly same heights.

Input

The first line contains two integers: nnn and mmm (1≤n≤105;1≤m≤2⋅105)(1\le n\le 10^5; 1\le m\le 2\cdot 10^5)(1≤n≤105;1≤m≤2⋅105) – the number of positions and the number of queries.
Next mmm lines contains the information of queries by following formats:

  • If the ithi^{th}ith query is type 1, the ithi^{th}ith line contains three integers: 1, pip_ipi​, and hih_ihi​ (1≤pi≤n,1≤hi≤10)(1\le p_i\le n,1\le h_i\le 10)(1≤pi​≤n,1≤hi​≤10), where pip_ipi​ is the position of the new tree and hih_ihi​ is the initial height of the new tree.
  • If the ithi^{th}ith query is type 2, the ithi^{th}ith line contains two integers: 2 and xix_ixi​ (1≤xi≤10)(1\le x_i\le 10)(1≤xi​≤10), where the xix_ixi​ is the index of the tree we want to cut.
    The input is guaranteed to be correct, i.e.,
  • For type 1 queries, pip_ipi​ will be pairwise distinct.
  • For type 2 queries, xix_ixi​ will be less than or equal to the current number of trees.
  • At any time no two trees have the exactly same heights.

In each line integers are separated by single spaces.

Output

Print mmm integers – the length of the longest increasing subsequence after each query. Separate the numbers by whitespaces.

Translation

有一条有nnn (n≤105)(n\le10^5)(n≤105)个位置的路，现有mmm (m≤2⋅105)(m\le 2\cdot10^5)(m≤2⋅105)个操作，分为两种：

  1. 在第ppp (p≤n)(p\le n)(p≤n)个位置种下一棵高度为hhh (1≤h≤10)(1\le h\le 10)(1≤h≤10)的树
  2. 砍掉从左向右第xxx (1≤x≤10)(1\le x\le10)(1≤x≤10)棵树

每个单位时间内，树会长高111米。
每次操作后，输出当前最长上升子序列的长度。
数据保证任意时刻没有两棵树高度相同。

Solution

由于nnn比较大，显然不能用二维树状数组/线段树维护DP\mathrm{DP}DP。

考虑最长上升子序列如何DP\mathrm{DP}DP。从后往前DP\mathrm{DP}DP，每次找到后面的数值比当前大的位置，选出其中DP\mathrm{DP}DP值最大的加111，作为当前位置的DP\mathrm{DP}DP值。找后面符合条件的最大DP\mathrm{DP}DP值可以用线段树维护。如果将位置和数值作为x,yx,yx,y值放到坐标系上，发现如果将x,yx,yx,y值对换后，问题是完全相同的。于是可以得到另一种DP\mathrm{DP}DP方法：从数值最大的向数值最小的DP\mathrm{DP}DP，每次找到比其大的数值中对应位置在其后面的数对应的DP\mathrm{DP}DP值的最大值，将最大值加111作为当前数的DP\mathrm{DP}DP值。同样地，这种方法也可以用线段树维护。

观察题目中的特殊条件，发现h,xh,xh,x均不大于101010。如果插入一棵高度为hhh的树，则需要将所有高度小于hhh的树的DP\mathrm{DP}DP值重新计算；如果砍掉第xxx棵树，则需要将第1∼(x−1)1\sim (x-1)1∼(x−1)棵树的DP\mathrm{DP}DP值重新计算。每次重新计算的位置均不大于101010个。

考虑维护两棵线段树，分别对应前面的两种DP\mathrm{DP}DP方式（第一棵按xxx从大到小算，第二棵按yyy从大到小算）。种树时，找出所有高度小于hhh的位置，将第一棵线段树中对应位置的DP\mathrm{DP}DP值清零，按高度从大到小排序，然后依次计算DP\mathrm{DP}DP值，由于DP\mathrm{DP}DP到每个位置时，比它高的树都已经被DP\mathrm{DP}DP过了，所以可以保证正确性。砍树时，找出第1∼(x−1)1\sim(x-1)1∼(x−1)棵树，将第二棵线段树中对应高度的DP\mathrm{DP}DP值清零，从后向前依次计算DP\mathrm{DP}DP值，正确性同样可以保证。注意修改DP\mathrm{DP}DP值时在两棵线段树上都需要修改。
为了快速找到前(x−1)(x-1)(x−1)棵树，可以维护一个堆，存储当前有树的位置，每次弹出前(x−1)(x-1)(x−1)小即可。

总时间复杂度O(10×nlog⁡n)O(10\times n\log{n})O(10×nlogn)。
